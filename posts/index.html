提問的智慧 提問前的嘗試 在 Google 輸入關鍵字，瀏覽網站的文章或影片，理解其中的內容。 提問前的心態 問題可能不會立刻被回覆，迅速被解決。 提問時的內容 使用 gist 或 carbon 提供原始碼，說明程式碼某些行數，根據 Google 理解的內容，你預期結果會出現什麼，實際結果又是什麼。如果是一個專案的話，就把它放到 GitHub 上，然後給予我特定檔案的連結喔。 提問之後 說聲謝謝，並且吸收這些解答者給予的解答，另外給予答覆。然後將得到解答的那個討論串請務必註記「已解答」。

用 Hugo 撰寫筆記 前言 寫筆記不用擔心誤人子弟，筆記主要是寫給自己看的，從中得知自己的不足；其次也能透過分享，讓別人揪出你的錯誤。 工程師習慣用 Markdown 撰寫筆記，有眾多平台都有支援 Markdown 語法，例如 Hugo、HackMD、Notion、docsify 等等，而我最後選擇了 Hugo 主要有兩個原因： 對比 HackMD，有些筆記較適合以 pdf、ppt 的方式呈現，存在本地較易連結文件。 對比 Hexo，Hugo 建置網頁時間快速許多。 開始使用 Hugo 上手 Hugo 十分簡單，只要照著官網操作即可，我選擇的主題是 PaperMod。 如果要建立 GitHub Pages，Hugo 跟 Hexo 不同，需要搭配 GitHub Action，根目錄才會產生 index.html，操作請參考 Create A Blog WIth Hugo And Deploy To Github In 10 Minutes。 題外話，假設你剛起步，只需要一個網站，單單使用 GitHub Pages 即可，但若是你有「很多網站」，或是需要更多「進階」功能 ，Netlify 是個不錯的選擇，可以嘗試看看。 安裝套件 Google Analytics 同一個 GA 帳號下能擁有數個資源，而每個資源底下可以透過不同的檢視條件，篩選出數個資料檢視。免費版的 GA 帳號目前提供最多 50 個資源，每個資源最多可以有 25 種資料檢視。 GA 左上角會顯示目前的資源，然後參考 Google Analytics 4 in Hugo，但我嘗試後仍未成功。...
